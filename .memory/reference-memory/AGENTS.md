# reference-memory

本目录用于存放引用记忆，只记录“去哪里看什么”的入口索引，不记录正文结论。

这个目录是模板目录，新增引用记忆时应优先基于本目录已有分类模板填写，避免每次临时决定字段和结构。

## 读取规则

1. 第一轮只读取每个文件前 30 行的 `YAML front matter`。
2. 只有在确实命中当前任务时，才展开正文。
3. 正文只写索引入口、用途说明和使用边界，不写最终结论。

## 分类模板

- `template.md`
  - 通用引用记忆模板。
- `source-reference.md`
  - 源码、仓库、设计实现参考模板。
- `api-reference.md`
  - API 文档、调试地址、接口入口模板。
- `script-reference.md`
  - 脚本、命令入口、运维入口模板。

## 维护约束

1. `decision_policy` 固定使用 `index_only`。
2. 这里只存入口索引，不存总结结论。
3. 若某条入口已经沉淀为稳定规则，应回写正式设计文档，而不是停留在引用记忆中。
